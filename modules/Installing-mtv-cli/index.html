OpenShift Cluster Managed Bare Metal Documentation
  • Migration Toolkit for Virtualization
This site uses Just the Docs, a documentation theme for Jekyll.

You can install the {mtv-full} from the CLI.

Prerequisites

  • OpenShift CLI installed

  • Admin privileges

Procedure

  1. Log in to the MTV cluster and create an openshift-migration project:

    $ cat << EOF | oc apply -f -
---
apiVersion: project.openshift.io/v1
kind: Project
metadata:
  name: openshift-migration
EOF

  2. Create an OperatorGroup object called migration:

    $ cat << EOF | oc apply -f -
---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: migration
  namespace: openshift-migration
spec:
  targetNamespaces:
    - openshift-migration
EOF

  3. Create a Subscription object for the virt-operator:

    $ cat << EOF | oc apply -f -
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: virt-operator
  namespace: openshift-migration
spec:
  channel: development
  installPlanApproval: Automatic
  name: virt-operator
  source: konveyor-for-vms
  sourceNamespace: openshift-marketplace
  startingCSV: "konveyor-virt-operator.v99.0.0"
EOF

  4. Create a VirtController object:

    $ cat << EOF | oc apply -f -
---
apiVersion: virt.konveyor.io/v1alpha1
kind: VirtController
metadata:
  name: virt-controller
  namespace: openshift-migration
spec:
  olm_managed: true
EOF

  5. View the resources created by the VirtController:

    $ oc get pods -n openshift-migration
